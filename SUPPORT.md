# Support

Thanks for using Void! This document explains where to ask questions, how to report problems, and what to expect.

## Questions and help
- Start with the README and examples in the `test` module.
- For general questions:
  - GitHub Discussions
  - StackOverflow with the `kotlin` tag and a link to this repository

## Bugs and feature requests
- Search existing issues first to avoid duplicates
- Open a new issue with:
  - Void version, JVM version, OS
  - Steps to reproduce and minimal code sample
  - Expected vs. actual behavior

## Security issues
Please do not file security-related issues publicly. Follow the process in [SECURITY.md](SECURITY.md).

## Response expectations
Void is an open-source project maintained by volunteers. We try to respond to new issues within a few days. Complex topics may take longer. PRs that include failing tests demonstrating a bug usually get faster attention.

## Commercial support
No official commercial support is provided. If you are interested in sponsoring development, please open an issue to discuss details.
